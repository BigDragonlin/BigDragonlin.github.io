# C#与Lua交互原理?？

Wrap文件：每一个Wrap文件都是对一个C#类的包装。

C# Call Lua交互过程 C#文件先调用Lua的解析器底层的dll库（C语言编写），再由DLL文件执行相应的Lua文件

Lua Call C# 交互过程 1.Wrap方式：首先生成C#源文件对应的Wrap文件，Lua文件会调用生成的Wrap文件，再由Wrap文件去调用C#文件。 2.反射方式：当索引系统API、DLL库或者第三方库，如果无法将代码具体实现进行代码生成，可通过反射来获取，执行效率较低。

C#与Lua交互原理：虚拟栈！！ 交互通过虚拟栈实现，栈的索引分为正数和负数，如果索引是正数，则1表示栈底，如果索引是负数，则-1表示在栈顶

C# Call Lua交互原理 C#先将数据放入栈中，然后Lua去栈中获取数据，然后返回数据对应的值到栈顶，再由栈顶返回至C#

Lua Call C#交互原理 C#源文件生成Wrap文件、或C#源文件生成C模块，将Wrap文件和C模块注册到Lua的解析器中，最后再由Lua去调用这个模块的函数~

从代码文件方面解释： lua调用C#过程： lua->wrap->C# 先生成Wrap文件（中间文件/适配文件），wrap文件把字段方法，注册到lua虚拟机中（解释器luajit），然后lua通过wrap就可以调C#了、或者在config文件中添加相应类型也可以

C#调用Lua过程： C#生成Bridge文件，Bridge调dll文件（dll是用C写的库），先调用lua中dll文件，由dll文件执行lua代码 C#->Bridge->dll->Lua 或 C#->dll->Lua

# 简单描述一下啊Lua的GC原理

垃圾收集器会在内存到达某阈值或者规定的步进或者手动启动垃圾收集器。 垃圾收集器会从根节点开始便利，将不可到达的节点标记为垃圾，然后清理垃圾节点， 释放内存。

# table的一些重要知识点有哪些

  1. table是lua中的一种数据结构，用来帮助我们创建不同的数据类型，如数组、字典等
  2. table是一个关联数组，可以用任意类型的值作为数组的索引，但是索引不能为nil，所有索引值都需要用 “[“和”]” 括起来； 如果是字符串，还可以去掉引号和中括号； 即如果没有[]括起，则认为是字符串索引，Lua table 是不固定大小的，你可以根据自己需要进行扩容；
  3. table的初始索引是从"1"开始的，如果没有写索引，默认从1开始，并按照顺序递增
  4. table的变量是一个地址引用，所以当你将一个table赋值给另一个table时，实际上是将地址赋值给了另一个table，所以当你改变其中一个table的值时，另一个table的值也会改变
  5. table不会固定大小，有新数据插入时会自动增长
  6. table可以保存任何类型数据，包括function和table

# 说说Lua中的闭包

闭包是什么：闭包是一个函数，加上该函数所引用的外部变量，一起构成了闭包 闭包的优点：可以访问外部函数的局部变量，可以保持外部函数的局部变量不被释放 使用lua实现简单的任务管理器

function createScheduler()
    local tasks = {} -- 更改变量名为复数形式，明确表示可以存储多个任务
    -- 创建任务
    function createTask(taskName, func) -- 修正拼写错误
        tasks[taskName] = {
            func = func,
            interval = interval,
            lastTime = os.time()
        }
    end
    -- 执行任务
    function runTask()
        for k, v in pairs(tasks) do -- 保持原有逻辑
            v.func()
            v.lastTime = os.time()
        end
    end
    return {
        createTask = createTask, -- 修正拼写错误
        runTask = runTask
    }
end
local myTask = createScheduler()
myTask.createTask('task1', function() -- 修正拼写错误
    print("task1")
end)
myTask.createTask('task2', function()
    print("task2")
end)
myTask.runTask()

# 深拷贝

适用于基本数据类型：number、string、boolean、nil 行为：直接复制值，不会影响原数据 示例：

local a = 10
local b = a
b = 20
print(a) -- 10

适用于复合数据类型：table、function、userdata、thread 行为：复制引用，会影响原数据 示例：

function copy_Table(obj)
    function copy(obj)
        if type(obj)  ~= "table" then						--对应代码梳理“1”  （代码梳理在下面）

        end
        local newTable={};									--对应代码梳理“2”

        for k,v in pairs(obj) do
            newTable[copy(k)]=copy(v);						--对应代码梳理“3”

# 说说lua中如何实现面向对象

Lua中table是非常强大的数据结构，利用table再结合元表metatable，可以方便的在Lua中模拟类，并实现面向对象编程中具有的特性：封装、继承、多态

  1. 利用lua实现类
Person = {}

function Person.new(name, age)
    local t = setmetatable({}, {__index = Person})
    t.name = name
    t.age = age
    return t
end

function Person:sayHello()
    print("Hello, my name is " .. self.name)
end

local p1 = Person.new("meng", 18)

# 解释以下lua中的元表何元方法

  1. 元表metetable：允许该表table行为，行为关联元方法，类似一种“操作指南”，包含各种操作行为的解决方案！！！
  2. 元方法：当表执行某些操作失败的时候，操作指南里的元方法指导你的行为~~
  3. __index：当访问表中不存在的键时，会调用该元方法
  4. __newindex：当给表中不存在的键赋值时，会调用该元方法
  5. __add：当对表进行加法运算时，会调用该元方法
  6. __sub：当对表进行减法运算时，会调用该元方法
  7. __mul：当对表进行乘法运算时，会调用该元方法
  8. __div：当对表进行除法运算时，会调用该元方法
  9. __mod：当对表进行取模运算时，会调用该元方法
  10. __unm：当对表进行一元负运算时，会调用该元方法
  11. __concat：当对表进行连接运算时，会调用该元方法
  12. __eq：当对表进行等于运算时，会调用该元方法
  13. __lt：当对表进行小于运算时，会调用该元方法
  14. __le：当对表进行小于等于运算时，会调用该元方法
  15. __call：当对表进行调用时，会调用该元方法
  16. __tostring：当对表进行转换为字符串时，会调用该元方法
  17. __metatable：当对表进行访问元表时，会调用该元方法
  18. __getmetatable：当对表进行设置元表时，会调用该元方法 ...
local father = {money = 100}
local son = {}
setmetatable(son, {__index = father})
print(son.money) -- 打印是100

# Lua中pairs何ipairs的区别？

  1. pairs：遍历表中所有的键值对，无序遍历
  2. ipairs：遍历表中所有的键值对，有序遍历，只能遍历数组,且键值对的键是数字，且是连续的,从1开始,直到nil为止
