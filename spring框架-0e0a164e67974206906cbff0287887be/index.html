            # Spring框架

            # 第一章 Spring概述

            # Spring框架是什么

            Spring是面向企业的轻量级java开发框架，它是为了解决企业应用开发的复杂性而创建。Spring的核心是AOP和IOC。

            # Spring框架的优点

            1. 轻量

              spring的核jar包只有3m左右，比较小

            2. 针对接口编程，解耦合

              Spring提供了控制反转，由容器管理对象。原来在代码中创建对象的方式，现在由容器创建完成。和对象之间解耦合。

            3. AOP编程的支持

              OOP和AOP编程，由传统的OOP编程到现在的AOP编程

              OOP编程 Object Oriented Programming的缩写，意为：面向对象编程

              AOP编程 AOP为Aspect Oriented Programming的缩写，意为：面向切面编程，通过预编译方式和运行期动态代理实现程序功能的统一维护的一种技术。

            4. 方便集成各种优秀的框架

              Spring不排斥各种优秀的开源框架，相反，Spring可以降低各种开源框架的使用难度了。Spring支持了各种框架的使用，Spring像插板，其它开源框架像插头，可以轻易的组合在一起。不需要的话可以轻易的移除。

            # 什么是控制反转（IOC）

            把原来直接操控类转移到由容器来操控，这样可以达到解耦和的效果

            Inversion of control 。依赖注入DI （dependency injection）

            依赖：classA中含有classB中的实例，称为classA对classB有依赖

            # 什么是AOP

            AOP（Aspect Orient Programing）

            Aspect ：切面，切面的特点，一般都是非业务功能。独立使用

            Orient：面向，对着

            Programing ：编程

            AOP的两种实现方式JDK的动态代理，CFLB的动态代理

            # 我们为什么要用AOP动态代理

            使用AOP可以把业务的各个部分隔离，解耦和，增加程序的复用性，提高编程效率。

            OOP：面向对象编程 Object Orient Programing

            # 怎样理解面向切面编程？

            1. 需要分析功能的时候找出切面
            2. 合理的安排切面执行的时间（在目标方法前，还是目标方法后）
            3. 合理的安排切面的位置，对哪个目标方法增加功能

            # AOP编程术语：

            1. Aspect：切面，表示增强的功能，解释一堆代码，完成一堆功能。非业务功能。常见的切面有日志，事务，统计信息，参数检查，权限验证
            2. JoinPoint：连接点。表示被切面植入的具体方法，通常业务接口中的方法均为连接点
            3. PointCut：切入点，表示连接点的集合
            4. target：目标对象，表示被增强的对象
            5. Advice：通知定义了代码的执行顺序，是目标代码前，还是目标代码后。

            一个切面有三个关键的要素

            1. 切面的功能代码
            2. 切面的执行位置
            3. 切面的执行时间

            aop的实现

            aop是一个规范，一个标准

            aop的技术实现框架：

            1. spring：spring在内部实现了aop规范，能做aop的工作

            2. aspectJ：一个开源的专门做aop的框架。sping框架中集成了aspectJ框架，通过spring就能实现aspectJ的功能。

              使用xml配置文件

              使用注解的方式实现aop

            # Spring的事务管理

            能够使用事务的特性来管理具体的业务

            # 事务管理器接口

            事务管理器接口PlantFormTransactionManager接口对象。

            主要完成事务的提交、回滚以及事务的状态信息。

            commit(TransactionStatus status)

            getTransaction(TransactionDefinition definition)

            rollback(TransactionStatus status)

            PlantformTransactionManager接口有两个实现类

            DataSourceTransactionManager:使用JDBC或者MyBatis进行数据化持久性操作时使用

            HibernateTranactionManager：使用Hibernate进行数据持久化时使用。

            Spring回滚

            spring默认回滚机制是：发生运行异常和error时回滚，发生受查（编译）异常时提交。不过可以我们也可以手动设置。

            受检查的异常应该使用try-catch方法捕获或者throw扔出

            不受检查的异常不要求被try-catch或者throw扔出

            # 五个事务的隔离级别

            脏读、幻读、不可重复读

            这五个常量都是以ISOLATION_***为开头

            1. DEFAULT :mysql默认的是repeat_read
            2. ISOLATION_UNCOMMTTED:读未提交
            3. ISOLATION_COMMITTED:读已提交
            4. ISOLATION_READ:可重复
            5. ISOLATION_SERIALIZABLE:串行化

            # 使用Spring的事务注解管理事务（掌握）

            通过@Transactional注解方式，将事务植入到要用事物的方法当中去，实现方法对事务的需求。

            @Transactional的可选属性如下

            1. propagation(n.传播，拓展，宣传)。用于设置事务的传播属性
            2. isolation：设置事务的隔离级别
            3. readOnly：用于设置事务是不是只读，如果在查询操作的时候，把这项设置为只读，可以增加程序的执行效率。
            4. timeout：超时，默认值为“-1”，就是不设置超时时间。
            5. rollbackFor:指定要回滚的异常类。类型为Class[]
            6. rollbackForClassName:需要回滚的异常类名称
            7. noRollbackFor:不需要回滚的异常类名字
            8. noRollbackForClassName:不需要回滚的异常类名字

            # 七个事务传播常量（propagation）

            传播规则回答了这样一个问题，就是一个新的事务应该被启动还是被挂起，或者是一个方法是否应该在事务性上下文中运行。

            1. PROPAGATION_REQUIRED

              就是让事务具有传染性，碰到就传染

            2. PROPAGATION_REQUIRES_NEW

              指定的方法支持当前新事务，但是当前没有事务也可以以非事务的方法执行

            3. PROPAGATION_SUPPORTS

              新建一个事务，如果当前存在事务，就把当前的事务挂起，直到新事物执行完毕
