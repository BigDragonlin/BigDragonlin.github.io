# Metadata

metadata是网页的自我介绍，title是搜索引擎抓的标题， description是搜索引擎抓的介绍。

# 在nextjs上添加metadata。

在最新版的nextjs上，直接在page.tsx中导出metadata 如下：

import type { Metadata } from 'next';

// 这就是我们要添加的“自我介绍”
export const metadata: Metadata = {
  title: '关于我们 - 我的公司名称', // 浏览器标签和搜索结果的标题
  description: '了解我们的团队、历史和使命。我们致力于提供最优质的服务。', // 搜索结果中的描述
};

// 这是你页面本来的内容
export default function AboutPage() {
  return (
    <div>
      <h1>关于我们</h1>
      <p>这里是关于我们页面的详细内容...</p>
    </div>
  );
}

# nextj和动态元数据

动态数据使用genarateMatadata

import type { Metadata } from 'next';

// 这是一个模拟函数，用来根据ID获取产品数据
async function getProduct(id: string) {
  // 在实际应用中，这里会是从数据库或API获取数据
  return {
    id: id,
    name: `产品 ${id}`,
    description: `这是关于产品 ${id} 的详细描述，它非常棒！`,
  };
}

// 注意！这里我们导出一个异步函数，而不是一个对象
export async function generateMetadata({ params }: { params: { id: string } }): Promise<Metadata> {
  // 1. 获取当前页面的ID
  const id = params.id;

  // 2. 根据ID获取这个页面的具体数据（比如产品信息）
  const product = await getProduct(id);

  // 3. 根据获取到的数据，动态生成并返回元数据
  return {
    title: `${product.name} | 我的商城`, // 例如："产品 123 | 我的商城"
    description: product.description,
  };
}

// 页面组件本身保持不变
export default async function ProductPage({ params }: { params: { id: string } }) {
    const product = await getProduct(params.id)
    return (
        <div>
            <h1>{product.name}</h1>
            <p>{product.description}</p>
        </div>
    )
}

# 总结

场景 使用方法 解释
静态页面 (如首页、关于我们) 导出一个 metadata 对象 标题和描述是固定的，直接写好就行。
动态页面 (如产品详情、博客文章) 导出一个 generateMetadata 函数 需要先获取数据（如产品信息），再根据数据生成标题和描述。
