# 命令模式概念:

命令模式允许将操作封装成对象，这种方式可以让你参数化其他对象的操作，或将操作排队执行，甚至支持撤销操作功能。这个模式主要涉及以下角色：

  • 命令接口（Command）: 声明执行操作的接口。
  • 具体命令（Concrete Command）: 实现命令接口，定义了接收者和行为之间的绑定关系。
  • 接收者（Receiver）: 知道如何实施与执行一个请求相关的操作。
  • 请求者（Invoker）: 要求该命令执行这个请求。
  • 客户端（Client）: 创建一个具体命令对象并设置其接收者。

# 实现步骤:

  • 定义命令接口: 该接口包含执行操作的方法。
  • 创建具体命令类: 实现命令接口，指定动作和接收者。
  • 实现接收者: 实际执行命令操作的类。
  • 创建请求者: 触发命令执行的类，比如按钮等。
  • 客户端配置: 设置具体命令，并关联请求者。

# 代码示例:

现在我们用一个简单的例子来实现命令模式，假设我们有一个灯光控制系统，可以开灯和关灯。

  • 命令接口（Command）: 包含一个execute()方法的接口。
  • 具体命令（Concrete Commands）: LightOnCommand和LightOffCommand，分别用于开灯和关灯。
  • 接收者（Receiver）: Light类，拥有on()和off()方法。
  • 请求者（Invoker）: 遥控器，有按钮用来执行命令。
  • 客户端（Client）: 设置命令并将其与按钮关联。
# include <iostream>
class Command {
public:
    virtual ~Command() {}
    virtual void execute() = 0; // 纯虚函数，使Command类成为抽象类
};

class Light {
public:
    void on() {
        std::cout << "灯打开了。" << std::endl;
    }
    void off() {
        std::cout << "灯关闭了。" << std::endl;
    }
};

class LightOnCommand : public Command {
private:
    Light& light;
public:
    LightOnCommand(Light& light) : light(light) {}
    void execute() override {
        light.on();
    }
};

class LightOffCommand : public Command {
private:
    Light& light;
public:
    LightOffCommand(Light& light) : light(light) {}
    void execute() override {
        light.off();
    }
};

class RemoteControl {
public:
    void submit(Command* command) {
        command->execute();
    }
};


int main() {
    Light light;
    LightOnCommand lightOn(light);
    LightOffCommand lightOff(light);

    RemoteControl remote;
    remote.submit(&lightOn); // 灯打开了。
    remote.submit(&lightOff); // 灯关闭了。

    return 0;
}
