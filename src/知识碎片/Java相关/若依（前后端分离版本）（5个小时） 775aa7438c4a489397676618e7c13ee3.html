# 生成验证码

基本思路：

  1. 前端默认访问80端口
  2. 前端默认执行create方法来问后端要验证码图片和uuid
  3. 后端执行 1+1=？@2
  4. 答案2放入redis，1+1=转成图片给前端

问题：

http://localhost/dev-api/login (opens new window)

前端验证码请求地址是这个，这个地址请求的是前端还是后端？

我们知道前端的端口是默认80，后端端口是8080，为什么我们要说这个网址请求的是后端呢？

使用的是代理，url请求前端，进行代理，映射到后端，解决跨域问题

//C:\Users\Administrator\Desktop\RuoYi-Vue\ruoyi-ui\vue.config.js
proxy: {
  // detail: https://cli.vuejs.org/config/#devserver-proxy

//发出所有带  VUE_APP_BASE_API 的请求，统统映射到 target，并且
//重写process.env.VUE_APP_BASE_API，替换为 空。
[process.env.VUE_APP_BASE_API]: {
   v: `http://localhost:8080`,
    changeOrigin: true,
    pathRewrite: {
      ['^' +process.env.VUE_APP_BASE_API]: ''
    }
  }
}
