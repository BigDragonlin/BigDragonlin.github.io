# SpringMVC

# 第一章 SpringMVC概述

# springMVC简介

是基于spring的一个模块；专门做web开发。可以理解为servlet的升级版。

web开发底层就是servlet，框架是在servlet基础上加些功能

springMVC就是一个spring。spring是容器。

springMVC中放的是控制器对象

servletmapping 编写 拓展名or 斜杠

# springMVC优点

  1. 基于MVC架构
  2. 容易理解，上手快，使用简单
  3. 作为Spring的一部份，可以使用ioc和奥胖。方便灵活，能够和mybatis，jpa等其它框架
  4. SpringMVC强化注解的作用，在控制器，Service，Dao中都可以使用，方便灵活。

# 对象参数接收

如果处理器方法有两个对象，且两个处理器对象的属性有重合，前端传回来的参数会给两个重复处理器对象属性赋一样的值。

# 返回ModelAndView

返回ModelAndView,我们需要在方法体中添加要处理的函数后通过ModelAndView.addObject添加要返回的数据，通过ModelAndObject.setView(“String”)，注册要吧ModelAndView传送过去的对象。返回ModelAndView，将ModelAndView交给StringMVC框架。

# 返回String

字符串代表着View视图，也就是返回地址，相当于forwode的地址

两种方式

  1. 处理器方法String，表示逻辑视图名称，需要配置视图解析器。
  2. 处理器方法放回String，表示完整视图路径；此时不能配置视图解析器。如果此时配置视图解析器，视图解析器会加上完整路径名称，这时我们再访问将不会得到正确的资源路径，导致报错404。

# 返回void（了解），和Ajax连起来使用

它不能表示视图，也不能表示数据。通过HttpServletResponde，可以通过printlin来传输数据。

缺点：代码有重复

  1. java对象转为json
  2. 通过HtteServletResponse给browser通信

# 返回对象Object

例如String，Integer，Map等。所以Object是数据，和视图无关。可以使用对象表示数据，响应ajax请求。

# 返回对象是String

区分返回对象是对象还是视图，看有没有ResponseBody，有的话就是数据，没有的话就是视图。

如果ajax的type设施为json，控制器返回的是文本，将会出现转化错误。

# 路径加斜杠和不加斜杠有什么区别

我们要用斜杠还是可以不用斜杠

我们在jsp，html中使用的地址，都是相对地址。

地址分类

  1. 绝对地址，带有协议名的都是绝对地址，http：//www.baidu.com

  2. 相对地址，没有协议开头的，例如user/some.do。相对地址不能独立使用，必须要有一个参考地址。必须要参考绝对地址加上相对地址，才能得到正确的响应地址。

  3. 参考地址

    当你的地址没有斜杠开头的时候，我们的访问地址就会编程，当前的地址加上你没有加斜杠的地址。

# 异常处理

# @ExceptionHandler注解

throw和throws的区别

区别一：throw 是语句抛出一个异常；throws 是方法抛出一个异常；

区别二：throws可以单独使用，但throw不能；

区别三：throw要么和try-catch-finally语句配套使用，要么与throws配套使用。但throws可以单独使用，然后再由处理异常的方法捕获。

异常发生处理逻辑 1,需要把异常记录下来，记录到数据库，日志文件。 记录日志发生的时间，哪个方法发生的，异常错误内容。 2,发送通知，把异常的信息通过邮件，短信，微信发送给相关人员。 3,给用户友好的提示。

  1. 在SpringMVC容器中创建 annocation-scan，再创建annocation-driver
  2. 创建MyException类，继承Exception和Exception（message）。NameException和AgeException集成Exception
  3. 创建GlobleException类，类上注解 ControllerAdvice。分别创建对饮NameException和AgeExcepion的方法，方法体上注解ExceptionHandler（Exception的类型）
