# pnpm 与 npm 的关系

# 关系定位

  • pnpm 是 npm 的替代品，不是替代者（仍需要 npm 生态）
  • pnpm 完全兼容 npm 的 package.json 和 npm registry
  • pnpm 可以直接替换 npm 命令，学习成本低

# 命令对照

npm 命令 pnpm 命令
npm install pnpm install
npm add react pnpm add react
npm run dev pnpm dev
npm test pnpm test

# pnpm 相比 npm 的核心优势

# 1. 磁盘空间节省（最显著优势）

# npm 的问题

项目A/node_modules/lodash/  (4.5MB)
项目B/node_modules/lodash/  (4.5MB)  # 重复存储
项目C/node_modules/lodash/  (4.5MB)  # 重复存储

# pnpm 的解决方案

~/.pnpm-store/
  └── lodash@4.17.21/  (4.5MB)  # 只存一次

项目A/node_modules/.pnpm/
  └── lodash@4.17.21 → 硬链接到全局存储

项目B/node_modules/.pnpm/
  └── lodash@4.17.21 → 硬链接到全局存储（不占额外空间）

# 实际效果

  • 100 个项目使用相同依赖，npm 占用 100 份空间
  • pnpm 只占用 1 份空间 + 极小的硬链接开销
  • 节省 70-80% 磁盘空间

# 2. 安装速度更快

# 性能对比（官方 benchmark）

场景：安装 React 应用依赖

工具 首次安装 有缓存时
npm 51s 15s
yarn 39s -
pnpm 24s ⚡ 1.5s ⚡⚡⚡

# 为什么更快？

  1. 并行下载：同时下载多个包
  2. 内容寻址存储：已下载的包直接硬链接，不重新下载
  3. 增量安装：只处理变更的依赖

# npm 的问题

// package.json 中没有声明 lodash

    "some-package": "1.0.0"  // 这个包依赖 lodash
  }
}

import _ from 'lodash'  // npm 允许，pnpm 禁止

危害：

  • 依赖隐式存在，未来 some-package 移除 lodash 会导致崩溃
  • 依赖版本不可控

# pnpm 的解决方案

node_modules/
├── some-package/  (符号链接)
└── .pnpm/
    ├── some-package@1.0.0/
    │   └── node_modules/
    │       └── lodash/      # 只在 some-package 内可访问
    └── lodash@4.17.21/

# 4. 更好的 Monorepo 支持

# npm workspaces vs pnpm workspaces

pnpm 方式（pnpm-workspace.yaml）：

packages:
  - 'apps/*'
  - 'packages/*'
1
2
3

pnpm 优势命令：

pnpm -r run build          # 递归运行所有包的 build
pnpm --filter @acme/ui build  # 只构建 ui 包及其依赖
pnpm --filter "./apps/**" test # 只测试 apps 下的所有包
1
2
3

npm workspaces 局限：

  • 缺少高级过滤功能
  • 依赖安装效率不如 pnpm

│   │   └── node_modules/
│   │       ├── react/
│   │       └── loose-envify/  # react 的依赖，项目访问不到
│   └── lodash@4.17.21/

# 对比总结

特性 npm pnpm
磁盘空间 重复存储 节省 70%+
安装速度（有缓存） 基准线 快 5-10 倍
幽灵依赖 存在 完全杜绝
依赖隔离 扁平化（松散） 严格隔离
Monorepo 基础支持 高级过滤
生态兼容 100% 99.9%
学习成本 - 极低

# 迁移建议

如果您的项目考虑从 npm 迁移到 pnpm：

# 1. 全局安装 pnpm
npm install -g pnpm

# 2. 删除旧依赖
rm -rf node_modules package-lock.json

# 3. 用 pnpm 安装
pnpm install

# 4. 更新脚本（如果使用 npm run）
# package.json 中的 scripts 无需修改
# 只需将命令从 npm run xxx 改为 pnpm xxx
