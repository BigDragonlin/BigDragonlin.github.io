C++智能指针是一种封装了原始指针的类，用于自动管理动态分配的内存，以避免内存泄漏。 C++标准库（STL）中提供了三种主要的智能指针：st d::unique_ptr、std::shared_ptr和std::weak_ptr，每种智能指针都有其特定的用途和管理策略。

# 正常指针释放过程

MyClass* my_class = new MyClass();
my_class->classMethod();
delete my_class;
my_class = nullptr;
//指针为null
my_class->classMethod();

# std::unique_ptr

std::unique_ptr是一种独占所有权的智能指针。它保证同一时间只有一个unique_ptr可以指向给定的资源。 当unique_ptr被销毁时（例如，当它超出作用域时），它指向的对象也会被自动删除。

#include <memory>

class MyClass {
public:
MyClass() {}
void classMethod() {}
};

int main() {
std::unique_ptr<MyClass> myPtr(new MyClass());
myPtr->classMethod(); // 使用智能指针访问成员函数
}

# std::shared_ptr

std::shared_ptr是一种共享所有权的智能指针。多个shared_ptr可以指向同一个资源。 资源只有在最后一个指向它的shared_ptr被销毁后才会被释放。 这通过引用计数来实现，每次复制shared_ptr时计数增加，每次销毁时计数减少。

#include <memory>

class MyClass {
public:
MyClass() {}
void classMethod() {}
};

int main() {
std::shared_ptr<MyClass> myPtr1 = std::make_shared<MyClass>();
std::shared_ptr<MyClass> myPtr2 = myPtr1; // myPtr1和myPtr2共享所有权
myPtr2->classMethod();
}

# std::weak_ptr

std::weak_ptr是一种不控制对象生命周期的智能指针。 它设计用来解决shared_ptr相互引用时可能导致的循环引用问题。 weak_ptr需要与shared_ptr一起工作，通过weak_ptr可以观察但不延长shared_ptr管理的对象的生命周期。

#include <memory>

class MyClass {
public:
std::weak_ptr<MyClass> other; // 用weak_ptr解决潜在的循环引用问题
MyClass() {}
void classMethod() {}
};

int main() {
std::shared_ptr<MyClass> myPtr1 = std::make_shared<MyClass>();
std::shared_ptr<MyClass> myPtr2 = std::make_shared<MyClass>();

    myPtr1->other = myPtr2;
    myPtr2->other = myPtr1; // 创建一个循环引用

}

通过使用智能指针，可以有效地管理动态分配的内存，避免内存泄漏，简化内存管理操作。智能指针还提供了一些额外的安全和便利特性，如自动资源释放、异常安全等。
