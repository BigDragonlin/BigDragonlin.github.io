# git文件中文件的状态

Git工作区中的文件存在两种状态： untracked未跟踪（未被纳入版本控制） tracked已跟踪（被纳入版本控制） 1)Unmodified未修改状态 2)Modified已修改状态 3)Staged已暂存状态

# 本地仓库常用命令

# 将暂存区中的文件取消或者是切换到指定版本

# 将暂存区的文件提交到版本库
git commmit

# 远程仓库操作命令

## 查看远程仓库
git remote

## 查看远程仓库详细信息
git remote -v

## 添加远程仓库
git remote add origin标签 远程仓库url

## 从远程仓库克隆
git clone <url>

## 从远程仓库拉取
git pull <short-name> <branch-name>

## 如果本地仓库不是从远程仓库中克隆，而是从本地仓库中新建，并且本地仓库中存有文件，此时
git管理软件将会报错（fatal:refusing to merge unrelated histories）
这时候要
git pull --allow-unrelated-histories <short-name> <branch-name> 

## 推动到远程仓库
git push <remote-name> <branch-name>

# 分支操作

  • 分支是git中的一个重要的概念，使用分支意味着你的工作可以和主线工作分开，各个分支相互独立，互补干扰。
  • 通过git init命令我们可以创建一个默认master分支。
## 查看分支
git branch

## 创建分支
git branch [name]

## 切换分支
git checkout [name]

## 推送远程仓库分支
git push [shortName][name]

## 合并分支
git merge [name]

# 标签操作

git中的标签，是指在某一个时间，git仓库中数据的状态，类似于快照。通过标签，我们可以快速找到我们在某一个事件开发时候的状态。

标签起到版本控制的作用，分支起到开发隔离的作用

  • 标签相关命令
## 列出已有标签
git tag

## 创建标签
git tag [tag name]

## 将标签推送到远程库
git push [short_name] [tag_name]

## 检出标签
git checkout -b [branch] [tag_name]
