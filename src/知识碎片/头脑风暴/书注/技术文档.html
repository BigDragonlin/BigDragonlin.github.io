# 📄 知识共读平台 —— 技术开发文档（MVP）

项目代号：Marginalia
版本：v0.1（MVP）
目标：验证“用户上传正版 PDF → 哈希匹配 → 协作批注”核心流程

# 二、核心数据模型（Supabase）

# 1. books 表（文档群组）

字段 类型 说明
id UUID (PK) 文档群组唯一 ID
file_hash TEXT (UNIQUE) PDF 的 SHA-256 哈希（hex）
title TEXT 自动提取的标题
isbn TEXT 若识别到 ISBN（如 9787107265873）
verified BOOLEAN 是否通过 ISBN 官方验证
created_at TIMESTAMPTZ 首次上传时间

# 2. annotations 表（批注）

字段 类型 说明
id UUID (PK) 批注 ID
book_id UUID (FK → books.id) 所属文档群组
user_id UUID (FK → auth.users) 创建者
page_number INT 页码（从 1 开始）
type TEXT highlight, note
content TEXT 用户输入文本（≤200 字）
position JSONB { x, y, width, height }（PDF points 坐标）
visibility TEXT private, group（仅同群组可见）
created_at TIMESTAMPTZ 创建时间

# 3. users 表（Supabase Auth 自动创建）

后端逻辑：

  1. 保存 PDF 到 Supabase Storage（设置 5 分钟自动删除）；
  2. 用 pdfjs-dist 提取元数据（/Title）；
  3. 若无标题，调用 OCR（可选，MVP 可跳过）；
  4. 计算 SHA-256 哈希；
  5. 查询 books 表：
    • 若存在 → 返回 book_id；
    • 若不存在 → 插入新记录，verified = !!isbn；
  6. 删除临时 PDF；
  7. 返回结果。

🔒 安全：此接口需用户登录（Supabase Auth）。

# 四、前端核心逻辑（Next.js App Router）

# 1. PDF 上传与匹配流程

// app/upload/page.tsx
'use client';
const handleUpload = async (file: File) => {
  const formData = new FormData();
  formData.append('pdf', file);
  
  const res = await fetch('/api/upload', { method: 'POST', body: formData });
  const { book_id, title, copyright_notice } = await res.json();
  
  // 跳转到阅读页
  router.push(`/reader/${book_id}`);
};

# 2. PDF 渲染与批注（使用 PDF.js）

  • PDF 文件始终从用户本地加载（URL.createObjectURL(file)）；
  • 不从服务器下载 PDF；
  • 批注层叠加在 <canvas> 上；
  • 坐标使用 page.getViewport().convertToPdfPoint(x, y) 转换为 PDF 内部坐标。

# 3. 笔记同步

  • 保存时调用 Server Action：

    await saveAnnotation({ book_id, page_number, content, position });
    1

  • 加载时按 book_id + page_number 查询。

# 五、合规与安全设计

# 1. 版权提示（强制展示）

  • 在 /reader/[book_id] 页面顶部显示：

    📚 《{title}》
    本平台不提供电子书。请通过正规渠道获取正版。
    [官方购买链接] | [国家智慧教育平台]

# 2. 内容限制

  • 前端限制 content.length ≤ 200；
  • 禁止自动复制高亮原文（或截断至 20 字）；

# 3. 数据保留策略

  • Supabase Storage 中的 PDF：TTL = 5 分钟；
  • 用户可随时删除自己的批注；
  • 支持版权方通过 file_hash 申请屏蔽（预留 admin API）。
