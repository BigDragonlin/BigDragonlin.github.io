# 📄 知识共读平台 —— 技术开发文档（MVP）

项目代号：Marginalia
版本：v0.1（MVP）
目标：验证“用户上传正版 PDF → 哈希匹配 → 协作批注”核心流程
原则：不存储 PDF、不传播内容、版权风险最小化

# 一、系统架构概览

✅ 关键约束：PDF完全本地处理，前端计算哈希后传到后端匹配，用户本地保持PDF文件。

🛡️ 安全原则：全流程零存储，仅传输元数据，任何PDF内容不触网。

# 二、核心数据模型（Supabase）

# 1. books 表（文档群组）

字段 类型 说明
id UUID (PK) 文档群组唯一 ID
file_hash TEXT (UNIQUE) PDF 的 SHA-256 哈希（hex）
title TEXT 自动提取的标题
isbn TEXT 若识别到 ISBN（如 9787107265873）
verified BOOLEAN 是否通过 ISBN 官方验证
created_at TIMESTAMPTZ 首次上传时间

# 2. annotations 表（批注）

字段 类型 说明
id UUID (PK) 批注 ID
book_id UUID (FK → books.id) 所属文档群组
user_id UUID (FK → auth.users) 创建者
page_number INT 页码（从 1 开始）
type TEXT highlight, note
content TEXT 用户输入文本（≤200 字）
position JSONB { x, y, width, height }（PDF points 坐标）
visibility TEXT private, group（仅同群组可见）
created_at TIMESTAMPTZ 创建时间

# 3. users 表（Supabase Auth 自动创建）

# 七、MVP 范围（必须做 vs 暂不做）

功能 状态 说明
用户上传 PDF ✅ 仅用于哈希计算
自动提取标题 ✅ 优先 PDF 元数据
SHA-256 匹配 ✅ 服务端校验
PDF 本地渲染 ✅ 不从服务器加载
高亮 + 文字批注 ✅ 坐标精准
同哈希用户可见笔记 ✅ visibility = 'group'
OCR 封面识别 ❌ Phase 2
手写笔迹 ❌ Phase 2
GitHub 同步 ❌ 永不作为核心
区块链 ❌ 不采用

# 八、风险控制清单

  • [ ] PDF 文件不在服务器持久化
  • [ ] 哈希计算在服务端完成
  • [ ] 标题不可由用户编辑
  • [ ] 默认笔记 visibility = 'group'（非 public）
  • [ ] 用户协议明确免责条款
  • [ ] 提供版权方投诉入口（预留）
