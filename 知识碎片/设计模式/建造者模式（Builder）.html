建造者模式（Builder Pattern）是一种常用的软件设计模式，用于将一个复杂对象的构建与其表示分离，使得同样的构建过程可以创建不同的表示。这种模式特别适合于那些需要生成多种类型对象的情况，每种类型对象的创建过程都可能包含复杂的步骤。

# 建造者模式的组成：

  • 产品（Product）：最终要创建的复杂对象。
  • 建造者（Builder）：接口定义了创建产品各个部分的方法。
  • 具体建造者（Concrete Builder）：实现 Builder 接口，提供构建和装配产品各个部分的具体实现。
  • 指挥者（Director）：负责按特定步骤构造产品。

# 用 C++ 实现建造者模式

下面是一个例子，我们将构建一个简单的车（Car）对象，包括车轮、引擎和车身作为组成部分。

步骤 1: 定义产品类

#include <iostream>
#include <string>

class Car {
public:
    std::string wheels;
    std::string engine;
    std::string body;

    void specifications() {
        std::cout << "Body:" << body << std::endl;
        std::cout << "Engine:" << engine << std::endl;
        std::cout << "Wheels:" << wheels << std::endl;
    }
};

步骤 2: 定义 Builder 接口

class CarBuilder {
public:
    virtual ~CarBuilder() {}
    virtual void buildBody() = 0;
    virtual void buildEngine() = 0;
    virtual void buildWheels() = 0;
    virtual Car* getCar() = 0;
};

步骤 3: 创建具体建造者

class FerrariBuilder : public CarBuilder {
private:
    Car* car;
public:
    FerrariBuilder() { car = new Car(); }
    ~FerrariBuilder() { delete car; }

    void buildBody() override { car->body = "Sports"; }
    void buildEngine() override { car->engine = "V8"; }
    void buildWheels() override { car->wheels = "Sport wheels"; }
    Car* getCar() override { return car; }
};

步骤 4: 定义指挥者

class Director {
private:
    CarBuilder* builder;
public:
    void setBuilder(CarBuilder* newBuilder) {
        builder = newBuilder;
    }

    Car* getCar() {
        Car* car = builder->getCar();
        builder->buildBody();
        builder->buildEngine();
        builder->buildWheels();
        return car;
    }
};

步骤 5: 使用建造者模式

int main() {
    Car* car; // Pointer to a car object
    Director director; // Director that controls the building process
    
    FerrariBuilder ferrariBuilder; // Create a builder for Ferrari cars
    director.setBuilder(&ferrariBuilder); // Assign this builder to the director
    
    car = director.getCar(); // Director constructs the car
    car->specifications(); // Display the car's specifications

    return 0;
}

# 解释：

  • 产品类（Car）：定义了车的特征和展示方式。
  • 建造者接口（CarBuilder）：提供了创建产品不同部分的抽象方法。
  • 具体建造者（FerrariBuilder）：继承自 CarBuilder，实现了如何构建具体的 Ferrari 车型。
  • 指挥者（Director）：控制建造过程，通过 CarBuilder 接口方法来构建最终产品。

这样，建造者模式允许你分步骤构建复杂对象，并通过同一建造过程可以创建不同的产品。这种模式在需要生成具有复杂构造过程的不同类型对象时非常有用。
